# data/base — AI Summary Base Store

This directory holds the canonical, pipeline-managed AI summaries for every L2
document in the openwrt-docs4ai corpus.

## Layout

```
data/base/
  <module>/        ← one subdirectory per pipeline module
    <slug>.json    ← one JSON file per L2 document
```

The `<slug>` matches the L2 document filename without the `.md` extension.

### Modules

| Module | Source |
|--------|--------|
| `ucode` | C source files in `openwrt-condensed-docs/ucode/` |
| `procd` | Header API docs in `openwrt-condensed-docs/procd/` |
| `uci` | UCI schema docs in `openwrt-condensed-docs/uci/` |
| `luci` | JavaScript source in `openwrt-condensed-docs/luci/` |
| `openwrt-hotplug` | Hotplug event docs in `openwrt-condensed-docs/openwrt-hotplug/` |
| `wiki` | Wiki pages in `openwrt-condensed-docs/wiki/` |
| `openwrt-core` | Core reference docs |
| `luci-examples` | LuCI example code docs |

## JSON Schema

```json
{
  "slug": "<L2 filename without .md>",
  "module": "<module name>",
  "title": "<L2 title field>",
  "content_hash": "<sha256[:12] or null>",
  "ai_summary": "<3-5 sentences; sentence 1 comprehensive, remaining sentences are details>",
  "ai_when_to_use": "<1–2 sentences: specific OpenWrt use case>",
  "ai_related_topics": ["<exact symbol or concept name>", "..."],
  "generated_at": "<ISO 8601 timestamp>",
  "model": "<model string — see below>",
  "pipeline_version": "v12",
  "saved_at": "<ISO 8601 timestamp written by lib.ai_store when persisted>"
}
```

### Field purpose

| Field | Purpose now (AI-V1) | Planned value (AI-V2+) |
|-------|----------------------|-------------------------|
| `slug` | Stable lookup key matching L2 filename. | Allows future cross-store joins and rename migration tracking. |
| `module` | Partitions files by subsystem and avoids slug collisions. | Module-level model/policy overrides. |
| `title` | Human readability in PR review and quick audits. | Potential ranking/UX hint in web index. |
| `content_hash` | Stale detection for pipeline-generated entries; `null` pins human entries. | Foundation for stale dashboards and drift telemetry. |
| `ai_summary` | Primary dense retrieval snippet and LLM routing hint. | Candidate for quality scoring and multilingual variants. |
| `ai_when_to_use` | Intent routing signal for agent/tool selection. | Could power task-conditioned retrieval weighting. |
| `ai_related_topics` | Graph-style symbol adjacency hints. | Could build explicit symbol co-occurrence index. |
| `generated_at` | Provenance for when content was authored/generated. | Freshness audits and regeneration scheduling. |
| `model` | Provenance by source path (`manual`, API, migration, seed). | Per-model quality analytics and fallback policies. |
| `pipeline_version` | Schema/logic compatibility marker. | Future migration gate for breaking schema revisions. |
| `saved_at` | Timestamp of last persistence to storage. | Conflict resolution for override/base reconciliation. |

### `content_hash` semantics

| Value | Meaning |
|-------|---------|
| `"sha256[:12]"` | Pipeline-generated. Stale if it no longer matches the current L2 body hash. |
| `null` | Human-authored. Always considered valid; never auto-invalidated. |

### `model` values

| Value | Meaning |
|-------|---------|
| `"gpt-4o-mini"` | Generated by the script 04 API path |
| `"github-copilot-seeded"` | Written by GitHub Copilot during data store initialization |
| `"migrated-from-legacy-cache"` | Migrated from legacy `ai-summaries-cache.json` |
| `"manual"` | Human-authored directly in this file |
| `"manual-override"` | Human-authored override copied from base and then edited |

## Regenerating or Seeding a Summary Manually

Use the prompt below in any capable LLM chat interface.
Copy the L2 Markdown content, fill in the `MODULE` and `SLUG` placeholders,
and paste the JSON output directly into the appropriate `data/base/<module>/<slug>.json`.

### Manual generation prompt

```
You are a technical documentation assistant for the openwrt-docs4ai pipeline.
Given the following OpenWrt documentation page, produce a JSON object with these fields:

- "slug": "<SLUG>" (provided)
- "module": "<MODULE>" (provided)
- "title": the page's top-level heading
- "content_hash": null   ← always null for human-generated entries
- "ai_summary": 3-5 sentences. Sentence 1 must be a comprehensive overview of
  the entire page. Sentences 2-5 must provide details/clarifications and name
  exact functions, types, constants, files, or API methods from the document.
- "ai_when_to_use": 1–2 sentences describing a specific, concrete OpenWrt task
  that this page helps with. Provide enough context that an AI agent can decide
  whether to retrieve this page.
- "ai_related_topics": a JSON array of 4-10 exact symbol names, subsystem names,
  or closely related concept strings present in the document. No prose.
- "generated_at": current ISO 8601 timestamp
- "model": "manual"
- "pipeline_version": "v12"

Output only the raw JSON object. No markdown fences. No explanation.

Hard constraints:
- Do not invent symbols that are absent from the source text.
- Preserve exact symbol spelling and capitalization from the source text.
- Keep ai_summary factual and implementation-oriented, not marketing-style.

MODULE: <MODULE>
SLUG: <SLUG>

DOCUMENT:
<paste L2 markdown here>
```

## Notes

- Never delete files from `data/base/` unless the source L2 document has been removed
  from the corpus.
- To customise a summary without losing the pipeline-managed version, copy the file to
  `data/override/<module>/<slug>.json` and edit it there.
- The script 04 pipeline will automatically pick up new files added here and will
  skip any slug that already has a valid (non-stale) entry.
- For the maintained scratch-first helper, use
  `python tools/manage_ai_store.py --option review`.
- Use `--option promote` after reviewing scratch results, or `--option full` to run
  the full review-plus-promotion sequence in one command.
- For detailed manual equivalents and fallback procedures, use
  [docs/specs/v12/ai-summary-operations-runbook.md](../../docs/specs/v12/ai-summary-operations-runbook.md).
