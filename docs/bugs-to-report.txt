4. openwrt/luci: `luci-app-dockerman` likely reflects intentional dual-mode script behavior, not a clear upstream bug
Ready to file: no
Target repo: https://github.com/openwrt/luci
Upstream path: `applications/luci-app-dockerman/root/usr/share/rpcd/ucode/docker_rpc.uc`

Current assessment
- Do not file this as a LuCI bug based on current evidence.
- The file is script-shaped as well as module-shaped: it has a shebang, inspects `/proc/self/comm`, exposes a CLI test mode when `caller != 'rpcd'`, and only returns `methods` after that direct-execution branch.
- LuCI JavaScript consumers declare rpc objects such as `docker`, `docker.container`, `docker.image`, `docker.network`, and `docker.volume`, which supports the direct multi-object `return methods;` shape.
- The earlier warning is more likely a downstream validator context problem caused by compiling a module-shaped snippet in script mode.

Why it is not reportable right now
- The script appears intentionally dual-purpose, not obviously malformed.
- The local validator has now been updated to retry ucode compilation in module mode when top-level `return` is the only failure signal.
- This makes the LuCI item look like a pipeline validation-context issue, not a confirmed upstream defect.
