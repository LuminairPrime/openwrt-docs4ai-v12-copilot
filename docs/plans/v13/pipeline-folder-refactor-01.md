# Plan 006: Pipeline Workspace Rationalization

**Status:** Draft
**Date:** 2026-03-28
**Scope:** Directory structure, CI workflow, display prefix, symlink bug, test
alignment, documentation

---

## Goal

Unify all pipeline working state under a single run directory per execution.
Eliminate scattered repo-root directories (`staging/` as OUTDIR default, `tmp/`
as WORKDIR default). Remove the vestigial `PUBLISH_DIR` promotion step. Make
display prefixes dynamic. Fix the symlink deletion bug. Update docs to match.

## Non-Goals

- Do NOT rename `release-tree/` or `support-tree/` (published contract names).
- Do NOT change `sync_tree.py` CLI interface.
- Do NOT change `data/base/`, `data/override/` (persistent AI store).
- Do NOT change `release-inputs/` or `content/cookbook-source/` (repo inputs).
- Do NOT create new documentation files beyond updating existing ones.
- Do NOT create a source-repo root `llms.txt` (explicitly out of scope).
- Do NOT add a local promotion step (unnecessary complexity for local dev).

## Target Directory Layout

Every pipeline run — local or CI — writes to this structure:

```
tmp/
  pipeline-YYYYMMDD-HHMMSSz/           # one per run, UTC timestamp
    input/                              # WORKDIR: clones, intermediates
      repo-ucode/
      repo-luci/
      repo-openwrt/
      repo-manifest.json
      cross-link-registry.json
      .cache/                           # wiki cache etc.
      L1-raw/                           # extracted by 02x scripts
        wiki/
        ucode/
        luci/
        ...
      L2-semantic/                      # generated by stage 03
        wiki/
        ucode/
        luci/
        ...
    output/                             # OUTDIR: all publishable output
      L1-raw/                           # mirrored copy for inspection
      L2-semantic/                      # mirrored copy for inspection
      release-tree/                     # the public deliverable
        {module}/
          llms.txt
          map.md
          bundled-reference.md
          chunked-reference/
          types/                        # .d.ts files when applicable
        llms.txt
        llms-full.txt
        AGENTS.md
        README.md
        index.html
      support-tree/                     # internal support state
        raw/
        semantic-pages/
        manifests/
        telemetry/
      llms.txt
      llms-full.txt
      AGENTS.md
      README.md
      index.html
      repo-manifest.json
      cross-link-registry.json
      {module}/                         # flat module dirs
        llms.txt
      CHANGES.md
      changelog.json
      signature-inventory.json
```

**Key clarifications:**

- L1-raw and L2-semantic live under BOTH `input/` and `output/`:
  - `input/L1-raw/`: written by extractor scripts (02x). This is the
    authoritative source.
  - `input/L2-semantic/`: written by stage 03 from `input/L1-raw/`. This is
    the authoritative intermediate.
  - `output/L1-raw/` and `output/L2-semantic/`: copies assembled by stage 05a+
    into the output tree for inspection and diagnostics.
- `WORKDIR` always resolves to the `input/` subdirectory.
- `OUTDIR` always resolves to the `output/` subdirectory.
- Computed paths (`L1_RAW_WORKDIR`, `L2_SEMANTIC_WORKDIR`, `RELEASE_TREE_DIR`,
  etc.) derive from WORKDIR and OUTDIR exactly as they do today. No formulas
  change.

**CI layout:**

CI uses fixed paths (each run already gets a clean VM, so timestamps add no
value):

```yaml
WORKDIR: ${{ github.workspace }}/tmp/pipeline-ci/input
OUTDIR: ${{ github.workspace }}/tmp/pipeline-ci/output
```

**Local layout:**

When env vars are absent, `config.py` auto-computes timestamped paths. No
directories are created at import time — only path strings are computed.
`ensure_dirs()` creates them when a pipeline script calls it.

## Phase 1: Core Path Refactor — `lib/config.py`

### 1a. Add `create_run_dir()` utility function

Add a function (NOT called at import time) that creates a timestamped run
directory and returns the `(workdir, outdir)` tuple:

```python
import datetime

def create_run_dir(base="tmp"):
    """Create a fresh timestamped pipeline run directory.

    Returns (workdir, outdir) paths. Directories are created on disk.
    """
    ts = datetime.datetime.now(datetime.timezone.utc).strftime("%Y%m%d-%H%M%Sz")
    run_dir = os.path.join(base, f"pipeline-{ts}")
    workdir = os.path.join(run_dir, "input")
    outdir = os.path.join(run_dir, "output")
    os.makedirs(workdir, exist_ok=True)
    os.makedirs(outdir, exist_ok=True)
    return workdir, outdir
```

### 1b. Change WORKDIR / OUTDIR default resolution

Replace the current:

```python
WORKDIR = os.environ.get("WORKDIR", "tmp")
OUTDIR = os.environ.get("OUTDIR", "staging")
```

With:

```python
def _resolve_defaults():
    """Resolve WORKDIR and OUTDIR from env vars or fresh timestamped dir.

    When both WORKDIR and OUTDIR are set via environment, use them directly.
    When either is set, use it and fall back to legacy defaults for the other.
    When neither is set, compute a timestamped run directory. No directories
    are created — only path strings are returned. ensure_dirs() creates them.
    """
    w = os.environ.get("WORKDIR")
    o = os.environ.get("OUTDIR")
    if w or o:
        return (w or "tmp"), (o or "staging")
    ts = datetime.datetime.now(datetime.timezone.utc).strftime("%Y%m%d-%H%M%Sz")
    run_dir = os.path.join("tmp", f"pipeline-{ts}")
    return os.path.join(run_dir, "input"), os.path.join(run_dir, "output")

WORKDIR, OUTDIR = _resolve_defaults()
```

**Why this is safe:**

- At import time: only string computation, NO filesystem side effects.
- Python module caching means re-imports get the same module with the same
  timestamp.
- When env vars are set (CI, smoke tests, explicit local runs), behavior is
  identical to today.
- When env vars are absent (ad-hoc local runs), the user gets a fresh
  timestamped dir whose paths are printed by `ensure_dirs()`.

### 1c. No changes to computed paths or `ensure_dirs()`

`L1_RAW_WORKDIR`, `L2_SEMANTIC_WORKDIR`, `RELEASE_TREE_DIR`, `SUPPORT_TREE_DIR`,
`REPO_MANIFEST_PATH`, `CROSS_LINK_REGISTRY` all derive from `WORKDIR` and
`OUTDIR` using `os.path.join()`. They automatically point to the right places
after 1b.

`ensure_dirs()` continues to create `WORKDIR`, `OUTDIR`, `L1_RAW_WORKDIR`, and
`L2_SEMANTIC_WORKDIR`. No changes needed.

## Phase 5: Fix Symlink Deletion Bug — `lib/output_sync.py`

### 5a. Add a `_safe_remove_entry()` helper

Add near line 245 (before `_sync_recursive`):

```python
def _safe_remove_entry(path: Path) -> None:
    """Remove a filesystem entry, handling symlinks correctly.

    Symlinks are unlinked directly to avoid shutil.rmtree following them
    into a target tree.  Regular files are unlinked.  Directories are
    removed recursively.
    """
    if path.is_symlink():
        path.unlink()
    elif path.is_dir():
        shutil.rmtree(path)
    else:
        path.unlink()
```

### 5d. Add regression test

In the appropriate pytest file (e.g., `tests/pytest/pytest_08_output_sync_test.py`),
add a test:

```python
def test_sync_tree_removes_extraneous_symlink_to_directory(tmp_path):
    """Extraneous symlink-to-dir in destination should be unlinked, not rmtree'd."""
    src = tmp_path / "src"
    dst = tmp_path / "dst"
    target_dir = tmp_path / "real_target"

    src.mkdir()
    dst.mkdir()
    target_dir.mkdir()
    (target_dir / "important.txt").write_text("keep me")
    (src / "file.txt").write_text("source file")

    # Create extraneous symlink in destination pointing to a real directory
    symlink_path = dst / "stale_link"
    symlink_path.symlink_to(target_dir)

    sync_tree(src, dst, delete_extraneous=True)

    # Symlink should be removed
    assert not symlink_path.exists()
    assert not symlink_path.is_symlink()

    # Target directory must be UNHARMED
    assert target_dir.is_dir()
    assert (target_dir / "important.txt").read_text() == "keep me"

    # Source file should be synced
    assert (dst / "file.txt").read_text() == "source file"
```

**Platform note:** On Windows, creating symlinks may require Developer Mode or
elevated privileges. If the test environment cannot create symlinks, the test
should be skipped with `pytest.mark.skipif` or a try/except around the symlink
creation.
