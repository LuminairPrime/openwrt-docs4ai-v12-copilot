# Release Tree Contract (V6)

**Version:** V6  
**Status:** Active — V13  
**Predecessor:** [`docs/archive/v12/specs/release-tree-contract.md`](../archive/v12/specs/release-tree-contract.md) (V5a)

## V13 Additions Over V5a

| Addition | Description |
|----------|-------------|
| `cookbook/` module | Task-oriented guidance module; produced by `02i` ingest from `content/cookbook-source/` |
| Per-module `AGENTS.md` | Behavioral AI guidance per module; static overlay files from `release-inputs/release-include/{module}/AGENTS.md` |
| Visible provenance headers | Blockquote provenance block injected by `05a` into each `chunked-reference/` file |
| Routing metadata fields | Optional `routing_summary`, `routing_keywords`, `routing_priority`, `era_status`, `audience_hint` in L2 |
| `MODULE_DESCRIPTIONS` | Curated module descriptions in stage `06` override heuristic summaries in root `llms.txt` |

## Module Schema

Every module folder contains the following files:

| File / Directory | Role |
|-----------------|------|
| `llms.txt` | Per-module LLM routing index |
| `map.md` | Navigation map (semantic skeleton) |
| `bundled-reference.md` | Complete bundled reference; becomes sharding index for oversized modules |
| `chunked-reference/` | Per-topic Markdown files from L2 semantic pages |
| `chunked-reference/{topic}.md` | Individual topic file with visible provenance header (V6+) |
| `AGENTS.md` | Per-module AI behavioral guidance (V6+, static overlay, modules: cookbook, luci, ucode) |
| `types/` | Optional; present only for modules with TypeScript IDE schemas |
| `types/*.d.ts` | Module-specific TypeScript declaration files |

---

## Oversized Module Sharding

When a module exceeds the token limit for a single file:
- `bundled-reference.md` becomes an index pointing to `bundled-reference.part-{NN}.md` files
- Part files sit alongside `bundled-reference.md` in the module root
- `chunked-reference/` topic files are always present regardless of sharding

---

## Guaranteed Absent Items

The following must never appear in the published release-tree:

- `L1-raw/`, `L2-semantic/`, `staging/`, `tmp/`
- `*-skeleton.md`, `*-complete-reference.md`
- `cross-link-registry.json`, `repo-manifest.json`, `signature-inventory.json`, `changelog.json`, `CHANGES.md`
- Any `.meta.json` sidecar

| File | Role |
|------|------|
| `README.md` | Human-readable introduction to the published corpus |
| `AGENTS.md` | Agent navigation guide listing all modules; references per-module AGENTS.md files and era guide |

## Routing Ownership Records

Each record defines which stage produces and finalizes a given output artifact, along with overlay behavior and validation ownership.

### Root routing indexes

| Field | Value |
|-------|-------|
| Artifact path | `release-tree/llms.txt`, `release-tree/llms-full.txt` |
| Artifact family | Root routing indexes |
| Stage family | Router |
| Generated by | `06` |
| Finalized by | `06` |
| Inputs | `L2-semantic/` |
| Overlay / override | None |
| Independent rerun | Rerun `06` |
| Validation owner | `08` |

### Root companion docs

| Field | Value |
|-------|-------|
| Artifact path | `release-tree/AGENTS.md`, `release-tree/README.md` |
| Artifact family | Root companion outputs |
| Stage family | Publication companion generator → finalizer |
| Generated by | `05b` |
| Finalized by | `07` |
| Inputs | `L2-semantic/`, `cross-link-registry.json`, `release-inputs/release-include/` |
| Overlay / override | Overlay may override the generated root file with a source-controlled replacement |
| Independent rerun | Rerun `05b`, then `07` |
| Validation owner | `08` |

### Root web index

| Field | Value |
|-------|-------|
| Artifact path | `release-tree/index.html` |
| Artifact family | Root web entrypoint |
| Stage family | Finalizer |
| Generated by | `07` |
| Finalized by | `07` |
| Inputs | `release-tree/`, `release-inputs/release-include/` |
| Overlay / override | Overlay may replace the generated index with an override file |
| Independent rerun | Rerun `07` |
| Validation owner | `08` |

## Support Tree

`support-tree/` is internal pipeline staging. It is ephemeral and never published to any public surface.

| Path | Contents |
|------|----------|
| `support-tree/raw/` | L1-raw output |
| `support-tree/semantic-pages/` | L2-semantic output |
| `support-tree/telemetry/` | `changelog.json`, `CHANGES.md`, `signature-inventory.json` |
| `support-tree/manifests/` | `cross-link-registry.json`, `repo-manifest.json` |

## Gatekeeper Rules (V6)

All checks must pass before any cross-repo publish.

| Check | Assertion |
|-------|-----------|
| Root router exists | `release-tree/llms.txt` exists and is > 512 bytes |
| Root index exists | `release-tree/index.html` exists |
| Configured module set | The published module directories match the configured module list for the active pipeline contract |
| No legacy leakage | No `L1-raw/`, `L2-semantic/`, or `openwrt-condensed-docs` in `release-tree/` |
| Fixed schema | Every module directory has `llms.txt`, `map.md`, `bundled-reference.md`, `chunked-reference/` |
| Chunked non-empty | Every module's `chunked-reference/` has at least one `.md` file |
| No stale provenance | L2 must not contain `version`, `upstream_path`, or `original_url` frontmatter fields |
| Dead links | No broken internal relative links in release-tree `.md` files |

```text
release-inputs/

│   └── README.md

└── release-repo-include/     (release-repo-only overlay, reserved)
```
