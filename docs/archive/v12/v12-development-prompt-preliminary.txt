# Session Strategy

This project is likely to span multiple Agent Manager sessions due to subscription token quota limits. The preferred approach is checkpoint-gated sessions: scope each session to complete exactly one or two checkpoints rather than attempting to auto-proceed through all nine stages in a single run. This eliminates mid-checkpoint truncation entirely.

Session start procedure:

- Read `docs/EXECUTION-MAP.md`
- Read `docs/IMPLEMENTATION-STATUS.md`
- Read recent git log (`git log --oneline -20`)
- Re-orient on the current checkpoint
- Continue from the recorded next action

If a session ends mid-checkpoint due to quota exhaustion:

- Finish the current atomic file write cleanly; do not start a new multi-file refactor.
- Update `docs/IMPLEMENTATION-STATUS.md` with checkpoint status, files changed, validations run, blockers, and the exact next action and command to run.
- Commit if the worktree is in a valid, non-broken state.
- Stop. On resume, rehydrate from repo state — do not re-plan the project from scratch.

Do not use a chat-only sentinel as the primary resume mechanism. Repository state plus the status files are the resume mechanism.

To correctly process individual `.c` files, each file must be physically copied into an isolated temporary directory before invoking `jsdoc2md`. Implement this workaround in `02b` exactly as specified in the roadmap. Do not attempt alternatives.
